Local-first multi-agent knowledge system

Tài liệu chi tiết mô hình xử lý Agent của HiveMind MD

Tài liệu này mô tả cách dự án biến một chủ đề hoặc câu hỏi thành knowledge base Markdown, index vector, trả lời bằng RAG, tự làm mới dữ liệu khi thiếu tri thức và ghi nhận vòng tự cải thiện.

Backend: FastAPI, SQLite, Ollama, ChromaDB/JSON fallback Frontend: React, TypeScript, Vite, TailwindCSS Ngày tài liệu: 2026-05-10 Nguồn đọc: code trong workspace

Tổng quan dự án

HiveMind MD là hệ thống local-first dùng nhiều agent nhẹ để thu thập, chuẩn hóa, rút trích, kiểm định, ghi Markdown và index tri thức. Trọng tâm của dự án không phải fine-tune model, mà là orchestration, RAG, memory và audit trail có thể kiểm soát.

Mục tiêu chính

Nhận một chủ đề hoặc câu hỏi, tạo kho tri thức Markdown có metadata, nguồn tham khảo và vector chunks để hỏi đáp lại.

Phong cách vận hành

Ưu tiên chạy local với Ollama, SQLite và file Markdown. Nếu thiếu Ollama hoặc ChromaDB, hệ thống có fallback để vẫn hoạt động.

Điểm nổi bật

Agent logs có stage, score, runtime, input/output count; self-improvement ghi lesson, prompt version và workflow suggestion.

Ba workflow quan trọng nhất

Workflow Entry point Mục đích Kết quả
Build Knowledge POST /api/agents/build-knowledge Tạo một file Markdown tổng hợp từ nhiều nguồn. backend/knowledge/<category>/<slug>.md và vector chunks.
Chat/RAG POST /api/chat Định tuyến câu hỏi, retrieve chunks, trả lời bằng context đã index. Câu trả lời, related files, sources, chat history Markdown.

Hợp đồng xử lý của Agent

Tất cả agent nghiệp vụ kế thừa BaseAgent. Cơ chế chung là gọi execute(...), agent tự chạy run(...), và kết quả luôn được đóng gói theo envelope thống nhất. 

{
  "agent": "SearchAgent",
  "status": "success | failed",
  "message": "Mô tả ngắn kết quả",
  "data": {}
}

Luồng thành công

BaseAgent.success(message, data) trả về tên agent, trạng thái success, message và payload data.

Luồng lỗi

execute() bắt exception, log lỗi và trả failed(message). Vì vậy orchestrator có thể ghi nhận lỗi theo cùng format.

Danh sách agent nghiệp vụ

Agent Vai trò Input chính Output chính
SearchAgent Tạo keyword và tìm URL nguồn qua WebSearchService. topic, limit keywords, sources
CrawlerAgent Crawl nội dung nguồn, hỗ trợ local://, HTTP fetch, trafilatura, BeautifulSoup fallback. sources documents gồm raw text và status.
CleanerAgent Lọc document thành công, chuẩn hóa text bằng clean_text(). documents cleaned_documents
ExtractorAgent Rút summary, key points, concepts, architecture, technologies, risks bằng Ollama hoặc heuristic fallback. cleaned_documents, use_ollama extracted
CriticAgent Chấm độ tin cậy nguồn theo URL và độ dài nội dung. cleaned_documents critiques gồm trust score, level, reason.
ComposerAgent Ghép tri thức thành Markdown, ghi metadata SQLite, rebuild knowledge map. topic, category, sources, extracted, critiques markdown_file, absolute_path, metadata
IndexerAgent Chunk Markdown, tạo embedding, upsert ChromaDB hoặc JSON fallback. file_path, metadata indexed_chunks, backend
InputRouterAgent Xếp hạng folder tri thức phù hợp cho câu hỏi dựa trên token, category, aliases, title, trust. query, max_routes top_routes, primary_folder, confidence.
AnswerAgent Retrieve chunks, ghép context, gọi Ollama trả lời hoặc fallback evidence-based. question, route_paths, route_context answer, related files, sources, confidence, retrieval preview.

Logic từng bước

Bước Logic xử lý Dữ liệu chuyển tiếp Ghi chú
Search Gọi WebSearchService.search(topic, limit); nếu DDG lỗi thì tạo nguồn local://search/<slug>. sources gồm title, url, snippet. Nguồn thấp giá trị bị lọc qua is_low_value_source().
Crawl CrawlerService.crawl_many() chạy gather song song từng source. documents gồm raw_text, status. HTML được extract bằng trafilatura trước, BeautifulSoup sau.
Clean Bỏ document failed, chuẩn hóa whitespace/noise. cleaned_documents gồm clean_text và content_length. Các document rỗng không đi tiếp.
Extract Gọi Ollama light model để trả JSON; nếu lỗi parse hoặc Ollama unavailable thì fallback sentence/concept heuristic. extracted theo URL. Prompt yêu cầu JSON có summary, key points, concepts, process, technologies, risks.
Critic Tính trust score theo source URL và độ dài nội dung. critiques Score dùng để hiển thị độ tin cậy và filter ở framework/cleanup.
Compose MarkdownService.compose() gom summary, key points, concepts, risks, source list và metadata comment. File Markdown, metadata SQLite, knowledge map. File mặc định nằm ở knowledge/<category>/<slug>.md.
Index VectorService.index_file() chunk Markdown, embed từng chunk, upsert vector store. Vector records và số chunk indexed. Ollama embedding lỗi sẽ dùng hash embedding 384 chiều.

Flow Chat/RAG với auto-refresh

Flow chat do ChatKnowledgeService điều phối. Điểm hay của luồng này là không chỉ retrieve vector đơn giản: nó còn định tuyến câu hỏi vào folder tri thức phù hợp, đọc knowledge map, tự refresh khi thiếu dữ liệu và lưu lịch sử chat thành Markdown.

User Messagemessage, conversation_id, auto_refresh
ChatInputAgentNormalize input, estimate tokens
InputRouterAgentChọn folder/category phù hợp
AnswerAgentMap excerpt + vector search + context
AutoRefreshAgentNếu confidence thấp hoặc needs_refresh
ChatMemoryAgentLưu turn vào Markdown history

Logic trong ChatKnowledgeService

  1. Chuẩn hóa câu hỏi bằng normalize_whitespace(), ước lượng token trước/sau để ghi log.
  2. Gọi InputRouterAgent.execute(query, max_routes=5) để tìm folder tri thức có liên quan.
  3. Gọi AnswerAgent lần 1 với route_paths và route_context.
  4. Nếu auto_refresh=true và answer báo needs_refresh=true, gọi KnowledgeRefreshService.refresh().
  5. Sau refresh, gọi AnswerAgent lần 2 để trả lời bằng dữ liệu mới.
  6. Lưu câu hỏi, câu trả lời, related files, sources và updated files vào knowledge/_chat_history/*.md.

Logic trong AnswerAgent

Giai đoạn Mô tả
Knowledge map Đọc _knowledge_map.md và lấy excerpt liên quan câu hỏi để giúp model định hướng.
Vector query Embed câu hỏi, query top chunks, lọc match score thấp hoặc file không trả lời được như agent-framework.md, _knowledge_map.md, chat history, quarantine.
Route merge Nếu router chọn folder, AnswerAgent đọc trực tiếp các Markdown trong folder đó để bổ sung route matches.
Context build Ghép route context, knowledge map excerpt và selected chunks trong giới hạn ký tự.
Generation Gọi Ollama main model với prompt chỉ trả lời dựa trên context. Nếu không có response, dùng fallback evidence sentences.
Confidence Lấy score cao nhất trong selected matches. Nếu confidence thấp hoặc ít chunk thì báo needs_refresh.
Khi không có chunk phù hợp, AnswerAgent không bịa. Nó trả lời rằng kho tri thức chưa đủ dữ liệu, có thể kèm knowledge map excerpt và bật cờ needs_refresh=true.

backend/knowledge/<category>/<topic-slug>/
  index.md

    01-<source-title>.md

Trước khi ghi refresh mới, service xóa các Markdown cũ có metadata "refresh_mode": true, đồng thời xóa metadata và vector records tương ứng. Điều này giúp topic folder không tích tụ shard cũ.

Nội dung từng loại file

File Nội dung chính Mục đích với agent
index.md Mục đích, tóm tắt, ý chính, khái niệm, danh sách nguồn/shard, rủi ro, metadata. Agent đọc trước để hiểu tổng quan và quyết định shard nào cần dùng.

Agent Framework chuyên sâu

AgentFrameworkService là workflow nâng cao cho chủ đề cần xử lý theo domain. Nó chọn profile, chia nguồn thành lane, chạy worker agents có giới hạn song song, tổng hợp, lọc nguồn yếu và ghi manifest agent-framework.md.

PlannerChọn profile, focus areas, source limit
Search lanesMỗi focus tạo query riêng
Source QualityDedupe, rank trusted domain, reject low-value
RouterChia source vào worker packets
WorkersCrawl, clean, extract, critic từng packet
Synthesis/FilterGộp, lọc theo trust và length
Save/Index/MapGhi shard, manifest, vector, map

Profile domain

Profile Category mặc định Focus areas Trust threshold
general general overview, core concepts, workflow, risks 0.10
finance finance market context, instruments, risk/regulation, personal finance 0.20
economy economy vĩ mô, GDP/lạm phát, chính sách, lao động, triển vọng Việt Nam 0.35
programming programming architecture, implementation, API/tooling, testing/performance, security 0.15
ai ai model behavior, agent workflow, retrieval/memory, evaluation, operations 0.15

Chi tiết worker lane

Framework tạo tối đa 5 worker packets. Mỗi packet chạy CrawlerAgent, CleanerAgent, ExtractorAgent và CriticAgent. Concurrency bị giới hạn bằng asyncio.Semaphore(profile.max_parallel_workers). Chỉ một số worker đầu dùng LLM extraction theo profile.use_llm_workers; các worker còn lại có thể dùng fallback để giảm tải phần cứng.

Cách framework chấm và trả log

  • Mỗi stage log có agent, stage, score, runtime_ms, input_count, output_count.
  • Worker score dựa trên tỷ lệ cleaned document và average trust score.
  • Filter score dựa trên tỷ lệ kept URL nhân với trust trung bình.
  • average_score là trung bình theo agent scores đã group.

Self-Improvement Loop

Self-improvement chạy sau từng agent trong OrchestratorAgent._execute_agent() nếu bật SELF_IMPROVEMENT_ENABLED. Nó tạo audit trail, chấm điểm output, rút bài học, tạo prompt version và workflow suggestion. Mặc định hệ thống không tự apply prompt, không tự sửa workflow thật và không tự sửa code production.

Agent Resultinput, output, status, runtime
ScoreServiceRecord agent_runs, scoreboard
EvaluatorAgentScore 0-60, risk, suggestions
ReflectionAgentLưu lesson nếu score thấp
PromptOptimizerTạo prompt version inactive
WorkflowOptimizerTạo suggestion pending

Vector store

VectorService ưu tiên ChromaDB persistent collection hivemind_knowledge. Nếu import hoặc init ChromaDB lỗi, service ghi JSON fallback tại backend/vector_store/fallback_index.json. Mỗi chunk có id SHA-256 từ relative_file:chunk_index, metadata file path, sources và trust score.

Fallback, kiểm soát lỗi và an toàn

Web search fallback

Nếu DDG/duckduckgo package lỗi, service tạo nguồn local://search/<slug> với snippet cảnh báo. Pipeline vẫn tạo được note nhưng trust thấp.

Crawler fallback

Nếu trafilatura không extract được, BeautifulSoup lấy text sau khi bỏ script, style, nav, footer, aside, form.

LLM fallback

Extractor, Answer và self-improvement đều có logic heuristic/rule-based khi Ollama không phản hồi hoặc JSON không parse được.

Vector fallback

Nếu ChromaDB không khả dụng, hệ thống dùng JSON index với hash embedding và cosine similarity.

Path safety

Các service ghi/đọc file dùng ensure_inside() để giữ path nằm trong knowledge_path.

Không tự thay đổi nguy hiểm

Self-improvement mặc định không tự apply prompt, không tự apply workflow thật, không patch code production và không fine-tune.

Cleanup/quarantine

KnowledgeCleanupService scan các Markdown không thuộc system dirs, phát hiện duplicate bằng fingerprint URL/hash, phát hiện noise theo blocked source, độ dài ngắn hoặc trust thấp. Nếu không dry-run, file bị chuyển vào _quarantine/<timestamp>/... với suffix .disabled, metadata và vector records bị xóa, sau đó rebuild knowledge map.
